# Personal Site Frontend Plan

## 1. 目标

本阶段只做前端，不改动后端接口设计与后端业务逻辑。

前端目标是基于当前 `Hertz` 项目结构，在现有服务端模板体系中实现一套可用、可扩展、风格统一的个人小站前台页面。视觉方向参考 [SimonAKing/HomePage](https://github.com/SimonAKing/HomePage/tree/master)，但不直接搬运其源码与构建链，而是复用其整体气质、页面节奏和交互感觉。

核心设计关键词：

- 暗色背景
- 细网格
- 中央聚焦
- 柔和光晕
- 平滑动效
- 响应式布局
- 极简导航

---

## 2. 当前约束

结合当前项目现状，前端实现遵循以下约束：

- 保持单体项目结构，不引入新的前端工程化框架
- 不迁入参考仓库的 `gulp / pug / less / dist` 构建体系
- 继续使用当前项目内的 `templates/` 和 `static/` 目录完成页面组织
- 首页第一版尽量少依赖接口，优先完成视觉骨架
- 等页面结构稳定后，再逐步接入已有文章、分类、标签接口

这意味着当前最合适的路线是：

1. 先搭模板结构
2. 再做样式系统
3. 再补页面动画
4. 最后接真实数据

---

## 3. 总体页面范围

前端第一阶段包含 4 个核心页面：

### 3.1 Home

首页承担品牌展示与站点入口作用，是整个前端的视觉重心。

目标效果：

- 黑底全屏首屏
- 细网格背景
- 中央头像
- 中央视觉标题
- 一句副标题
- 底部导航
- 第二屏展示部分内容摘要

导航暂时只放：

- `Blog`
- `About`
- `GitHub`

首页下方内容模块建议：

- 最新文章 3 篇
- 当前学习路线
- 技术标签云

### 3.2 Blog

博客列表页负责承载文章浏览。

第一版内容：

- 文章标题
- 摘要
- 分类
- 标签
- 发布时间

布局方向：

- 延续首页暗色风格
- 增加内容密度
- 保持留白与秩序感

第一版先不急着做复杂筛选，只保留后续扩展空间。

### 3.3 Article

文章详情页强调阅读体验。

第一版结构：

- 顶部标题区
- 发布时间 / 分类 / 标签
- 封面图
- 正文阅读区
- 底部返回入口

设计要求：

- 阅读列宽适中
- 正文层级清晰
- 暗色背景下保证足够对比度

### 3.4 About

About 页以静态内容为主，用于表达个人信息与技术路线。

建议内容：

- 简介
- 技术栈
- 学习路径
- GitHub 入口
- 实习以来积累的关注主题

学习路线当前固定展示为：

- `Hertz -> Kitex -> Eino`

## 4. 视觉方向

本项目复用参考站点的不是源码，而是以下设计语言：

### 4.1 首屏构图

- 内容集中在页面中央
- 周围留出大面积呼吸空间
- 首屏强调“个人入口页”而不是传统博客首页

### 4.2 背景气质

- 深色基底
- 精细网格
- 轻微发光元素
- 柔和渐变氛围

### 4.3 导航方式

- 首页导航不抢主视觉
- 以简洁横向入口呈现
- 图标可以后补，第一版先做文字即可

### 4.4 动效原则

- 动效服务于氛围，不喧宾夺主
- 先做轻量级 CSS/JS 动画
- 第一版不直接上重型 WebGL 特效

### 4.5 响应式要求

- 桌面端为主设计
- 移动端同步适配
- 不允许把移动端留到最后补

## 5. 技术实现原则

### 5.1 目录体系

后续前端文件统一按以下结构组织：

```text
templates/
  pages/
    home/
      index.html
    blog/
      index.html
    article/
      detail.html
    about/
      index.html
  components/
    layout.html
    nav.html
    article-card.html

static/
  css/
    reset.css
    theme.css
    home.css
    blog.css
    article.css
    about.css
  js/
    home.js
    background.js
  images/
```

### 5.2 实现路线

优先采用以下方式实现视觉效果：

- CSS 变量统一主题色、边框、阴影、网格透明度
- CSS 渐变实现大部分背景氛围
- CSS 伪元素实现网格和光斑
- 少量原生 JS 负责入场动画与交互增强

### 5.3 暂不引入

当前阶段不引入：

- React / Vue
- Tailwind
- Webpack / Vite
- 第三方动画框架
- 重型 WebGL 背景库

理由很明确：当前目标是先把小站前台跑起来，并且与你当前 `Hertz` 学习路径保持一致，避免技术栈横向扩张。

## 6. 页面落地顺序

### Phase F1：首页静态版

先完成首页首屏骨架：

- 页面整体布局
- 中央头像区
- 站点标题
- 副标题
- 底部导航
- 第二屏占位区

本阶段不追求复杂数据，不做花哨交互，重点是把气质定住。

### Phase F2：首页背景与动效

在首页静态结构稳定后补充：

- 网格背景
- 柔和光晕
- 入场动画
- 悬停反馈
- 平滑滚动体验

### Phase F3：Blog 列表页静态版

构建博客列表页视觉结构：

- 页面头部
- 文章列表卡片
- 分类/标签信息展示
- 空状态和占位样式

### Phase F4：Article 详情页静态版

搭建文章详情页：

- hero 区
- 正文区
- 封面图区域
- 底部返回与扩展区

### Phase F5：About 页

完成个人介绍与学习路线展示。

### Phase F6：接入真实数据

等前台静态页面稳定后，再逐步接入已有接口：

- 首页最新文章
- 博客列表
- 文章详情
- 分类与标签展示

---

## 7. 数据接线策略

前端数据接入采用“由浅入深”的策略：

### 7.1 首页

第一版首页尽量弱依赖接口。

可选接入：

- 最新文章 3 篇

其余内容如副标题、学习路线、标签云可先写静态内容。

### 7.2 Blog 列表页

优先接入：

- `/api/articles`

展示字段：

- 标题
- 摘要
- slug
- 分类
- 标签
- 发布时间

### 7.3 Article 详情页

优先接入：

- `/api/articles/:slug`

展示字段：

- 标题
- 摘要
- 正文
- 封面图
- 分类
- 标签
- 发布时间

### 7.4 About 页

第一版可纯静态，不依赖后端。

---

## 8. 首页重点设计说明

首页是第一阶段最关键的页面，先把它做好，整个站点的风格就立住了。

### 8.1 首屏结构

推荐采用以下层次：

1. 背景层
2. 网格层
3. 光晕层
4. 中央内容层
5. 导航层
6. 第二屏内容层

### 8.2 中央内容层

包含：

- 圆形头像
- 站点名称
- 一句个性副标题

副标题建议偏技术风格，例如：

- `STDIN | Think >> /dev/mind`
- `Learning Hertz, Kitex and Beyond`
- `Record, Build, Iterate`

具体文案后续可再调整。

### 8.3 导航层

首页导航暂定为：

- `Blog`
- `About`
- `GitHub`

展示原则：

- 与主视觉保持足够距离
- 不抢视觉中心
- hover 有轻微发光反馈

### 8.4 第二屏内容层

建议以 3 个模块组成：

- Latest Posts
- Learning Path
- Focus Tags

这样既能承接首页，也方便后续接入真实内容。

---

## 9. 样式系统建议

建议先建立全站统一主题变量，避免页面样式后期分裂。

可统一抽到 `static/css/theme.css` 的内容包括：

- 背景色
- 主文字色
- 次级文字色
- 高亮色
- 边框色
- 卡片背景
- 阴影
- 网格线透明度
- 圆角
- 内容最大宽度

这样后面 `home.css / blog.css / article.css / about.css` 只负责页面特定样式，不重复定义基础主题。

---

## 10. 交互原则

前端交互以“轻、稳、克制”为原则。

建议保留的交互：

- 首屏元素渐显
- 导航 hover 效果
- 卡片 hover 浮起
- 页面 anchor 平滑滚动

建议暂缓的交互：

- 复杂粒子系统
- 重型 3D 特效
- 大量并行动画
- 复杂过滤器动效

原因是第一版更需要稳定可维护，而不是炫技。

---

## 11. 验收标准

前端第一阶段验收以页面完整度为主：

### 首页

- 具备完整首屏视觉
- 桌面端和移动端布局正常
- 导航可点击
- 背景和动效不影响可读性

### Blog

- 列表结构完整
- 文章卡片样式统一
- 可承接后端文章数据

### Article

- 正文区域清晰可读
- 封面图、标题、元信息层级明确

### About

- 信息结构完整
- 与首页视觉语言一致

---

## 12. 当前默认执行顺序

后续前端开发按以下固定顺序推进：

1. `layout.html`
2. `home/index.html`
3. `theme.css`
4. `home.css`
5. `background.js`
6. 首页联调与微调
7. `blog/index.html`
8. `article/detail.html`
9. `about/index.html`
10. 再逐步接入接口

---

## 13. 默认结论

当前最合理的前端路线是：

- 不迁移参考仓库的构建系统
- 复用其视觉方向与页面气质
- 先把首页做成
- 再依次完成 Blog、Article、About
- 最后接真实后端数据

这条路线最稳，也最适合你当前“先学会 Hertz 单体开发，再逐步迭代”的整体节奏。
