# Phase 12 - 真实数据接入、前端联调修复与阶段产出沉淀

## Goal

在 phase-11 完成阶段 A/B 首轮工程化收口后，本阶段继续围绕真实使用场景修复前后台联调问题：移除前端 mock 与本地存储兜底，修复博客列表/详情页真实 API 加载链路，补齐后台文章编辑与删除所需的接口路径，并将数据库内容从临时测试文章调整为更符合实习产出记录的内容结构。

---

## 本阶段完成项

### 1. 前台博客切换到真实后端数据

- `static/blog/js/api.js` 移除 mock 文章、mock 分类、mock 标签与 localStorage 兜底逻辑
- 博客列表、分类、标签、文章详情统一从 `/api` 后端接口读取
- 分类和标签缓存改为以后端返回的 `id/name/slug` 为准
- 上下篇导航改为调用真实接口：`GET /api/articles/id/:id/adjacent`
- 列表页和详情页在加载文章前先加载分类、标签元数据，保证分类名和标签名能够正确回填

### 2. 修复静态页面缓存与详情页路由

- 博客静态资源增加版本参数 `?v=2`，降低浏览器旧缓存影响
- 静态资源缓存中间件对 `/api/` 和 `/health` 返回 `no-store`，避免接口响应被当成静态资源缓存
- `serveSPA` 增加 HTML 页面回退能力，例如 `/blog/post/:id` 可回退到 `blog/post.html`
- `biz.Register(h)` 提前到 SPA catch-all 路由之前，避免 API 路由被静态兜底吞掉

### 3. 后台文章编辑与删除链路修复

- 新增后台文章详情接口：`GET /api/admin/articles/:id`
- 后台编辑文章时不再从分页列表中查找正文，而是按 ID 请求完整文章详情
- `PUT /api/admin/articles/:id` 在 handler 中从路径参数回填 `req.ID`
- `DELETE /api/admin/articles/:id` 改为直接读取路径参数构造删除请求
- 修复后台删除和编辑在 thrift 字段迁移后容易出现的 ID 绑定问题

### 4. 实习产出内容入库

- 清理前一版按 Day1/Day2 写成的流水账测试文章
- 根据学习轨迹和项目文档，将数据库内容重写为 4 篇“实习产出总结版”文章：
  - 从环境搭建到工程实践：实习第一阶段的后端学习产出
  - 个人小站阶段交付：从内容系统到后台管理的完整闭环
  - 工程化收口：配置拆分、测试补齐、Docker 交付与 Swagger 清理
  - 从 SDK 接口流程到 Kitex 微服务：下一阶段后端演进方向（草稿）
- 分类收敛为：`实习产出`、`项目实战`、`后端工程`
- 标签收敛为：`Go`、`Hertz`、`个人小站`、`后台管理`、`前后端联调`、`工程化`、`AI 协作`、`测试`、`SDK`、`权限认证`、`Kitex`、`微服务` 等

**说明**：数据库内容已通过临时种子脚本写入，脚本执行完成后已删除临时 `cmd/seed-testdata` 目录，避免保留一次性工具代码。

### 5. 工程卫生修复

- 修复 `.gitignore` 编码异常，恢复为 UTF-8 文本
- 补充忽略本地 Go 编译产物：`*.exe`、`*.exe~`
- 修复 `service/article.go` 中因编码波动出现的中文响应消息乱码
- 保留本地编译产物在工作区外，不纳入提交
