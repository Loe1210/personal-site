## 目标

确定个人小站在单体阶段的开发基线，包括：

1. 项目目录规范
2. 开发文档规范
3. 第一批核心 IDL 初稿

## 完成内容

1. 明确项目当前走 `Hertz 单体 + proto 驱动核心接口` 的路线
2. 建立了项目规范文档
3. 起草了第一版 `article.proto`
4. 起草了第一版 `auth.proto`
5. 起草了第一版 `upload.proto`
6. 保留了已有的 UI 设计文档、后端规划文档和 IDL 规划文档

## 设计决策

### 1. 为什么现在就使用 proto

因为项目后续明确会往 `Kitex` 演进，所以希望在单体阶段就先固定核心领域契约，避免后面整体返工。

### 2. 为什么登录不用库内置 LoginHandler

因为当前项目希望统一接口风格，并让认证模块也能纳入 IDL 驱动链路，所以登录逻辑改为自己在 handler 中实现，JWT 只负责签发与校验。

### 3. 为什么不按模块开分支

因为真实开发会跨 `idl/handler/service/dal/router` 多层同时修改，按模块分支会让合并和 review 变得混乱。当前改为按阶段或功能闭环开分支。

## 遇到的问题

1. 当前工作区还没有正式的项目骨架，很多目录只是规划状态
2. 还没有最终确定 `go mod` 名称，因此 proto 中的 `go_package` 先使用了项目内路径占位，后续初始化项目时需要统一调整
