# 个人小站 UI 设计文档

## 1. 项目目标

这个个人小站有两个核心目标：

1. 做出一个有记忆点的首页，整体气质参考 `tomotoes.com`、`SimonAKing/HomePage`，以及你新确认的“中心头像 + 暗色网格”方向
2. 做出一个真正能长期承载内容的个人站，核心入口只保留 `Blog`、`About`、`GitHub`

这个站不是普通博客模板的简单套壳，而是一个“先有氛围，再进内容”的个人技术站。

## 2. 产品定位

这个站点应该是：

- 你的个人学习沉淀空间
- 你学习 `Go`、`Hertz`、`Kitex`、后续 `Eino` 的公开记录
- 一个可以从单体逐步演进的长期项目

这个站点不应该是：

- 信息很重的内容门户
- 多栏堆砌的资讯页
- 一上来就功能很多的管理系统

## 3. 信息架构

顶层导航暂时固定为：

- `Blog`
- `About`
- `GitHub`

建议的页面结构：

- `/` 首页
- `/blog` 文章列表页
- `/blog/:slug` 文章详情页
- `/about` 关于页
- `/github` GitHub 跳转页，或者直接外链

## 4. 设计关键词

- 极简
- 有氛围感
- 技术感
- 克制
- 安静
- 首屏聚焦强
- 轻微动态
- 像个人工作台

避免：

- 很像通用后台模板
- 过于花哨的插画风
- 页面内容很满
- 彩色渐变太多
- 屏幕上同时出现太多卡片
- 首页像普通博客落地页

## 5. 视觉方向

### 5.1 整体气质

整体风格建议接近“深夜工作台”与“终端桌面”：

- 接近纯黑的深色背景
- 很淡的网格线
- 中心单点聚焦
- 冷色微光
- 极少量装饰像素块或提示元素
- 轻微暗角与屏幕感

首页不是传统意义上的横幅，而是一个完整的“主场景”。

### 5.2 颜色系统

建议先用这一套色值：

```text
--bg-0: #040506
--bg-1: #090b0d
--bg-2: #0f1318
--grid-0: rgba(255, 255, 255, 0.05)
--grid-1: rgba(255, 255, 255, 0.025)
--surface-0: rgba(255, 255, 255, 0.04)
--surface-1: rgba(255, 255, 255, 0.07)
--line-0: rgba(255, 255, 255, 0.08)
--line-1: rgba(255, 255, 255, 0.14)
--text-0: #f3f5f7
--text-1: #c7ccd3
--text-2: #848b94
--accent-0: #7cf29a
--accent-1: #90e0ff
--accent-2: #d9dde3
--glow-0: rgba(124, 242, 154, 0.18)
```

使用规则：

- 背景以近黑色为主，不要偏蓝太重
- 网格线必须很淡，只作为气质层
- 绿色小高亮只做点缀，不做大面积主色
- 正文不要使用纯白
- 链接和交互态可以偏冷白或浅青

### 5.3 字体系统

不要直接使用很普通的默认字体栈。

建议方向：

- 英文标题：`Space Grotesk`、`Sora`、`Plus Jakarta Sans`
- 中文正文：`Noto Sans SC`
- 标签和少量技术标识：`JetBrains Mono`

字号建议：

```text
首页主标题：60-76px 桌面端，34-42px 移动端
首页副标题：18-22px 桌面端，15-17px 移动端
首页导航标签：14-18px
区块标题：28-36px
正文：15-17px
辅助信息：12-14px
```

排版规则：

- 首页主标题居中显示
- 副标题允许使用等宽字体或半等宽风格
- 文案必须短，首页不出现大段说明
- 下方入口名称应均匀分布，整体保持秩序感

## 6. 动效与背景

### 6.1 首页背景

首页首屏建议只保留一种主要动态背景：

- 深色网格背景
- 叠加非常轻的光晕流动
- 可选少量像素块闪烁或缓慢漂移

优先级：

1. 氛围感
2. 性能稳定
3. 首屏聚焦
4. 移动端可降级

### 6.2 动效规则

- 页面进入：整体淡入
- 头像：轻微呼吸感阴影或外圈亮度变化
- 导航入口 Hover：图标或文字上提 2px 到 4px
- 网格背景：低速移动或亮度波动
- 光标经过页面时，可选极轻的视差反应

避免：

- 很明显的大面积流体动画
- 过度拉扯的切换效果
- 页面中很多元素同时动来动去
- 一眼看上去像游戏启动页

## 7. 布局规范

### 7.1 全局布局

桌面端内容宽度建议：

- 首页中心内容宽度：`520px - 760px`
- 内容页最大宽度：`960px - 1120px`

间距节奏：

```text
8 / 12 / 16 / 24 / 32 / 48 / 72 / 96
```

### 7.2 响应式断点

```text
移动端：<= 767px
平板：768px - 1023px
桌面端：>= 1024px
宽屏：>= 1440px
```

规则：

- 首页在任何设备上都要保持“完整首屏”的感觉
- 首页核心内容始终垂直居中
- 小屏幕时入口改成三列紧凑排布或纵向排列
- 不要在移动端保留过多装饰像素元素

## 8. 页面级需求

### 8.1 首页 `/`

目标：

- 形成第一眼记忆点
- 告诉用户这是谁的站
- 自然引导进入 `Blog`、`About`、`GitHub`

页面结构：

1. 全屏主场景
2. 中心头像与标题区
3. 中心副标题区
4. 底部三入口导航
5. 极少量氛围装饰

首页不再强调传统顶部导航，也不做第二屏入口卡片。首页本身就是导航。

#### 8.1.1 首页主场景

构图要求：

- 整体为深色全屏
- 背景为淡网格
- 中间放头像、名字、签名
- 下方一排入口均匀排开
- 页面边缘可以带轻微暗角

这个首页应该让人第一眼联想到：

- 个人工作台
- 黑色终端桌面
- 克制的技术主页

#### 8.1.2 中心内容区

内容顺序：

1. 圆形头像
2. 主标题：你的名字
3. 副标题：一句身份描述
4. 可选补充说明：极短一句

建议文案结构：

```text
你的名字
Go 后端学习者 · 持续公开构建中
从 Hertz 出发，逐步走向 Kitex 与 Eino。
```

视觉要求：

- 头像是第一视觉点
- 主标题大小明显高于其他元素
- 副标题可以更像终端标签或状态行
- 补充说明如果保留，长度必须非常短

#### 8.1.3 首页导航入口

导航入口固定为：

- `Blog`
- `About`
- `GitHub`

入口形式建议：

- 图标在上，文字在下
- 或者纯文字配极简线性图标
- 三个入口水平居中分布

行为建议：

- 默认低对比显示
- Hover 时亮起并轻微上移
- 点击后进入对应页面或外链

### 8.2 Blog 列表页 `/blog`

目标：

- 清晰展示文章
- 优先保证可读性，而不是复杂视觉

页面结构：

1. 页面头部
2. 筛选区
3. 文章列表
4. 分页

头部建议：

- 标题：`Blog`
- 简介：一句话说明这里记录什么内容

筛选区：

- 标签或分类
- 默认按最新排序

文章卡片内容：

- 标题
- 摘要
- 标签
- 日期
- 预计阅读时间

视觉规则：

- 单列文章流完全可以接受
- 如果做双列，要保证足够留白
- 摘要保持短，不要大段堆叠
- 内容页可以延续首页的深色体系，但排版优先级高于装饰

### 8.3 文章详情页 `/blog/:slug`

目标：

- 让长文阅读舒适

页面结构：

1. 文章头部
2. 正文
3. 可选目录
4. 上一篇/下一篇导航

头部信息：

- 标题
- 发布时间
- 标签
- 可选导语

阅读规则：

- 优先打磨排版和间距
- 代码块要有清晰但克制的区分
- 图片展示要自然嵌入页面
- 阅读区不要被首页那种强装饰风格干扰

### 8.4 About 页 `/about`

目标：

- 介绍你自己
- 展示你的学习路线
- 让站点更有个人感

建议区块：

1. 简短自我介绍
2. 当前技术栈与兴趣
3. 学习路线
4. 为什么做这个站
5. 联系方式或外链

学习路线模块建议写成：

```text
现在：Hertz
接下来：Kitex
之后：Eino
```

视觉建议：

- 比首页更平静
- 使用大块留白与少量分割线
- 不要塞太多卡片

### 8.5 GitHub `/github`

有两个做法：

1. 直接跳转 GitHub 主页
2. 做一个简单过渡页，展示简介和几个仓库

第一版建议直接跳转即可。

## 9. 核心组件规范

### 9.1 首页入口组件

首页入口代替了传统导航，是最重要的交互组件。

要求：

- 图标极简
- 标签清晰
- 水平间距均匀
- Hover 态明显但克制

### 9.2 按钮

内容页和后续管理页至少准备三种：

- 主按钮：实体或发光描边
- 次按钮：Ghost 样式
- 文本按钮：纯文字链接

形状：

- 有圆角，但不要过软
- 推荐圆角：`12px - 16px`

### 9.3 卡片

卡片需要克制使用。

适合卡片的地方：

- Blog 文章预览
- About 页重点信息块

首页不建议使用传统玻璃卡片布局。

### 9.4 标签

标签风格建议：

- 小号胶囊
- 低对比背景
- 略带技术感
- 可少量使用等宽字体

### 9.5 页脚

页脚尽量简单：

- 一句收尾
- GitHub 链接

## 10. 内容文案风格

文案整体应当：

- 简洁
- 冷静
- 有一点思考感
- 略带技术气质

避免：

- 过于夸张的口号
- 首页自我介绍太长
- 首页出现大段说明文字
- 每个区块都写得很满

## 11. 可访问性与基础质量

最低要求：

- 文字对比度足够
- 键盘焦点可见
- 为重动效提供降级
- 移动端点击区域足够大
- 标题层级语义正确

## 12. 对 Hertz 的实现建议

建议的渲染方式：

- 首页和内容页可以先走服务端模板渲染
- Blog 内容从数据库或静态内容读取
- GitHub 用普通外链即可

建议模板结构：

```text
templates/
  layout/
  pages/
    home/
    blog/
    about/
  partials/
```

建议静态资源结构：

```text
static/
  css/
  js/
  images/
  fonts/
```

首页实现建议：

- 背景网格优先使用 CSS 生成
- 装饰块和暗角也尽量用 CSS，不依赖大图片
- 首页入口图标可以用 SVG 精灵或轻量图标库

## 13. 第一版范围

必须有：

- 中心头像式首页
- 暗色网格背景
- 首页入口：`Blog`、`About`、`GitHub`
- Blog 列表页
- Blog 详情页
- About 页
- 响应式布局

加分项：

- 背景轻微动态
- 头像轻微呼吸感
- 首页入口 Hover 动效
- 标签系统
- 预计阅读时间

第一版不需要：

- 评论
- 搜索
- 明暗主题切换
- 很重的后台视觉
- 首页复杂二级区块

## 14. 转成 Figma 的建议

如果后续要在 Figma 里重画，建议至少建立这些画板：

1. `首页 / 桌面端 / 1440`
2. `首页 / 移动端 / 393`
3. `Blog 列表 / 桌面端 / 1440`
4. `文章详情 / 桌面端 / 1440`
5. `About / 桌面端 / 1440`

首页画板应重点体现：

- 中心头像
- 居中文字层级
- 底部三入口
- 淡网格背景
- 暗角与极少量装饰元素

图层分组建议：

- `背景`
- `网格`
- `装饰元素`
- `头像`
- `标题`
- `副标题`
- `首页入口`
- `页脚`

本地样式建议：

- 颜色样式
- 字体样式
- 间距规范
- 图标尺寸规范
- Hover 状态规范

## 15. 最终设计原则

首页应该给人的感觉是：

`这是一个认真学习、认真构建的人做出来的个人技术主页。`

内容页应该给人的感觉是：

`内容清晰、值得读、之后还会想再来。`
